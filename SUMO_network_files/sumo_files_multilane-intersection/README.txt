sumo_files_multilane-intersection
==================================
SUMO network files for the ECO-AND multilane 4-arm intersection scenario.
This is the ACTIVE network used in all final ECO-AND experiments.

NETWORK
-------
simple-4-way.net.xml        Road network: 4-arm intersection, 3 lanes per arm,
                            straight-through traffic only (no turns).
tl.xml                      Traffic light timing override: 2-phase signal,
                            25 s green per direction (GGGrrrGGGrrr / rrrGGGrrrGGG).
                            Generated by SUMO netedit.
simple-4-way-gui.xml        SUMO GUI display settings (colours, zoom level).
simple-4-way-viewport.xml   Saved camera viewport for the SUMO GUI.
View-for-snipp.xml          GUI view preset for screenshots/snippets.

SIMULATION CONFIGS (.sumocfg)
------------------------------
interfering-vehicles-no-turns.sumocfg   ** ACTIVE ** Used by SUMO_Init.m.
                                        Straight-through traffic, no turns.
interfering-vehicles.sumocfg            Alternative config allowing turns.
free-flow.sumocfg                       Earlier single ego-vehicle config
                                        (used in legacy RL experiments).

Note: route-files and additional-files are left empty in the .sumocfg because
they are passed dynamically by SUMO_Init.m via the TraCI command-line call.

E3 DETECTOR FILES
-----------------
e3Detectors_150m.add.xml    Multi-entry-exit (E3) detectors placed 150 m from
e3Detectors_300m.add.xml    the junction stop line on each of the 4 arms (one
e3Detectors_450m.add.xml    entry + one exit per lane). These define the AV
                            control zone boundary (DSRC range).
                            SUMO_Init.m selects the correct file based on the
                            'range' variable (e.g. range=300 -> e3Detectors_300m).
                            Entry position = (road_length - range); exit at -0.1 m.

DATA COLLECTION
---------------
edgeLaneAggregate.add.xml   Defines two SUMO data collectors loaded every run:
                              laneData    -> laneDataDump.xml
                                            (waitingTime, laneDensity, speed per lane)
                              laneEmission -> laneEmissionDump.xml
                                            (fuel_abs, fuel_normed, fuel_perVeh,
                                             CO2_abs, CO2_normed, CO2_perVeh per lane)
                            Both files are written to the SUMO_results/ output
                            folder and parsed by readAllData.m after the run.

ROUTE FILES (route_files/)
--------------------------
flow-vph_200.rou.xml        ** ACTIVE ** Full MPR sweep (0-100%) at 200 veh/hr/lane.
flow-vph_400.rou.xml        ** ACTIVE ** Full MPR sweep at 400 veh/hr/lane.
flow-vph_600.rou.xml        ** ACTIVE ** Full MPR sweep at 600 veh/hr/lane.
flow-vph_800.rou.xml        ** ACTIVE ** Full MPR sweep at 800 veh/hr/lane.
                            Each file contains AV (type AV.STRAIGHT) and HDV
                            flows for all MPR levels in a single SUMO run.
                            Vehicle IDs encode type: HDV vehicles contain 'HDV'
                            in their ID; AVs do not.

flow-vph_*-MPR_25-50-75-100.rou.xml   Partial MPR variants (earlier experiments).
add0-ego / add1-horizontal / ...       Early single-vehicle and directional flows
                                       (used in legacy RL experiments).
vph200/, vph400/, vph600/, vph800/    Per-MPR individual route files: one file
                                       per (vph, MPR) combination. Used with the
                                       single-MPR route variant in SUMO_Init.m.
periodic_flow/                         Probability-based flow variant (not active).

PYTHON UTILITIES
----------------
generateTLSE3Detectors.py   SUMO tool: auto-generate E3 detector XML from the
generateTLSE2Detectors.py   network file for a given detection range.
                            Run these to regenerate e3Detectors_<range>m.add.xml
                            if the network geometry changes.

xml2csv.py                  Post-processing scripts: convert SUMO XML output
xml2csv_edgelanedump.py     files to CSV format. Useful for manual inspection
xml_to_csv.py               outside of MATLAB.
aggxml2csv2.py
csvpy.py
xsd.py                      XSD schema utility.
